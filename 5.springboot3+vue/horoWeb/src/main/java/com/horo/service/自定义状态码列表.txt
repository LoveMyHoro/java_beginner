1.响应状态码命名特点
    1xx（信息性状态码）：主要用于传递一些临时性的信息，表明请求已被接收，正在处理之中。这类状态码相对较少用到，比如常见的 100 Continue，它告知客户端可以继续发送剩余的请求内容，因为服务器已经接收到了请求的初始部分并且愿意接受后续部分来完成整个请求的处理。
    2xx（成功状态码）：表示请求已成功被服务器接收、理解并处理。这是我们希望在正常访问网页等操作时经常看到的一类状态码，例如 200 OK，意味着服务器成功处理了请求并且返回了所请求的资源内容；201 Created 常用于表示请求成功并且服务器创建了新的资源，比如通过 POST 请求成功创建了一条新的记录等。
    3xx（重定向状态码）：表明客户端需要进一步的操作才能完成请求，通常是需要进行页面的重定向。比如 301 Moved Permanently 表示所请求的资源已经永久性地移动到了新的位置，客户端后续应该使用新的 URL 来访问该资源；302 Found（或者 302 Moved Temporarily）则表示资源临时移动到了新位置，客户端此次访问应临时转向新的 URL。
    4xx（客户端错误状态码）：这类状态码指出请求出错的原因在于客户端。比如 400 Bad Request，往往是因为客户端发送的请求格式不正确，可能是请求参数有误、语法错误等导致服务器无法正确理解请求内容；403 Forbidden 表示服务器理解了请求，但拒绝执行该请求，可能是因为客户端没有足够的权限访问特定资源；404 Not Found 是大家非常熟悉的状态码，意味着客户端所请求的资源在服务器上不存在。
    5xx（服务器错误状态码）：表示请求失败的原因在于服务器端出现了问题。例如 500 Internal Server Error，这是一个比较笼统的服务器内部出错的状态码，可能是服务器程序运行出错、数据库连接故障等多种原因导致无法正常处理请求；502 Bad Gateway 通常是指服务器作为网关或代理时，从上游服务器接收到了无效的响应，导致无法正确传递给客户端；503 Service Unavailable 一般是指服务器暂时无法提供服务，可能是因为服务器正在维护、过载等原因。
